# GPTBest 中转服务 API 能力说明

## 📊 API 数量 vs 实际可用性

### 你看到的 500+ 个 API 接口

**这些接口包括**：
- ✅ 官方 OpenAI API（ChatGPT、GPT-4、Dall-E 等）
- ✅ 官方 Google API（Gemini、Claude 等）
- ✅ 官方 Anthropic API（Claude）
- ✅ 官方 Sora 2 API（视频生成）
- ✅ 其他第三方 API（Midjourney、Stable Diffusion 等）
- ✅ 工具类 API（翻译、语音、图片处理等）

### ❌ 但不是所有都能用

**关键限制**：
1. **需要对应的账户/密钥**
   - 你有 Sora 2 的密钥 → 可以用 Sora 2 API
   - 你没有 ChatGPT 的密钥 → 不能用 ChatGPT API
   - 你没有 Midjourney 的密钥 → 不能用 Midjourney API

2. **需要对应的配额**
   - 即使有密钥，也需要有足够的配额
   - 配额用完了就不能继续用

3. **需要对应的权限**
   - 某些 API 可能需要特殊权限
   - 某些 API 可能在你的地区不可用

## 🔑 如何使用其他 API？

### 步骤 1：获取 API 密钥

**以 ChatGPT 为例**：
1. 登录 OpenAI 官网（https://platform.openai.com）
2. 进入 API Keys 页面
3. 创建新的 API Key
4. 复制密钥

### 步骤 2：在应用中配置

**在应用设置中**：
1. 打开 **⚙️ 设置**
2. 选择 **"API 接口配置"** 标签页
3. 选择对应的 API 提供商
4. 填入 Base URL 和 API Key
5. 点击"测试连接"验证

### 步骤 3：在应用中使用

配置完成后，应用就可以调用该 API 了。

## 💡 常见误解

### ❌ 误解 1：有 500+ 个 API 就能用所有模型
**正确理解**：
- 500+ 个 API 只是中转服务支持的接口列表
- 你需要有对应的密钥才能使用
- 就像餐厅有 100 道菜，但你只能点你能付钱的菜

### ❌ 误解 2：配置一个密钥就能用所有功能
**正确理解**：
- 每个 API 都需要单独配置
- Sora 2 密钥只能用于视频生成
- ChatGPT 密钥只能用于文本生成
- 不能混用

### ❌ 误解 3：中转服务会自动转换 API
**正确理解**：
- 中转服务只是代理，不会转换
- 你用 Sora 2 API 就只能生成视频
- 你用 ChatGPT API 就只能生成文本
- 不能用 Sora 2 API 生成文本

## 📋 GPTBest 中转服务的实际作用

### ✅ 中转服务做什么

1. **提供统一的接口**
   - 所有 API 都用同一个 Base URL
   - 不用记住各个官方 API 的地址

2. **提供中文支持**
   - 中文文档
   - 中文客服
   - 中文界面

3. **提供多线路**
   - 美国线路
   - 香港线路（你现在用的）
   - 其他地区线路

4. **提供配额管理**
   - 统一的配额查询
   - 统一的使用统计

### ❌ 中转服务不做什么

1. **不会自动转换 API**
   - 你还是需要各个官方的密钥

2. **不会增加功能**
   - 只是代理，功能和官方一样

3. **不会降低成本**
   - 价格和官方一样（或更贵）

4. **不会解决所有问题**
   - 如果官方 API 不支持，中转也不支持
